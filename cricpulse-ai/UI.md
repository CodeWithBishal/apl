[UI/UX DESIGN PROMPT - "CRICPULSE AI" PLATFORM]

**Project Goal:** Design the interface for an AI-powered, real-time cricket co-host platform for IPL fans. The experience must be immersive, responsive, and prioritize two-way interaction.

**Core Design Elements:**

1.  **Visual Style & Theme:**
    - **Primary Vibe:** "Stadium Night Mode". A dark theme with vibrant, glowing accents (e.g., neon blue, electric yellow, or IPL team colors) to mimic the energy of a night match under floodlights.
    - **Background:** Use a subtle, deep gradient (`#0A0C10` to `#141824`). Abstract geometric patterns that evoke a cricket field (e.g., 30-yard circle) are a plus.
    - **Typography:** A modern, highly readable sans-serif for data (e.g., Inter, Poppins). Use a sporty, slightly bold font for key stats and the scorecard.

2.  **Layout Structure (Desktop-First, Mobile-Responsive):**
    - **Left Rail (25%): PERSONALIZED DASHBOARD**
        - Top: User profile with selected favorite team/player.
        - Below: Live stats feed (e.g., "Kohli's strike rate vs spin").
        - Bottom: Key match predictions ("Win Probability", "Required Run Rate").
    - **Center (50%): THE MATCH & CO-HOST**
        - Top: Animated, large-font scorecard (Team Score, Overs, Wickets).
        - Middle: **The Avatar/Waveform.** A subtle, animated AI icon or a soundwave visualizer that lights up when `CricPulse` is "speaking" or "listening".
        - Bottom: A scrolling **live commentary ticker** also shows AI's real-time observations.
    - **Right Rail (25%): COMMUNITY HUB**
        - Top: **"Fan Pulse" Meter.** A circular gauge or animated bar that changes color (Red=Angry, Yellow=Neutral, Green=Happy) based on real-time sentiment analysis.
        - Middle: **In-App Chat Room.** A typical chat interface with a text input box.
        - Bottom: **Active Users & Leaderboard.**

3.  **Interactivity & Micro-Animations:**
    - **Voice Input:** A prominent microphone button (pulsing animation when active) to allow hands-free queries.
    - **Live Sentiment:** When the `Fan Pulse` meter changes, the chat room border or the AI's avatar subtly flashes with the corresponding color.
    - **Key Event Burst:** When a wicket or boundary occurs, the scorecard should have a "shake" or "glow" animation, and the chat input box could suggest a prompt (e.g., "Was that out?").
    - **Loading States:** Skeleton screens for real-time data fetching.

4.  **Accessibility & UX:**
    - All actions must be possible with voice commands for a true "hands-free" experience.
    - Keyboard navigation is supported.
    - High color contrast for all text and stats.
